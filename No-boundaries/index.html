    No Boundaries

    2 minute read

    I recently was updating my Linkedin profile, and therefore received several phone calls from some recruiters.

    It’s good to be in conversations where people are showing interest in you, and it all went like breeze.

    There was one particular question that I didn’t think too much at the time, but echo in my mind after and I’d like to share some thoughts on it.

    Recruiter: Are you front end or back end?

    Me: (Without hesitation) I’m full stack.

    This made me think not because I take pride in being full stack (at least not the only reason), but how quickly I accept labelling myself as XX developer.

    The categories system in the question has definitely evolved from ten years ago.

    In the current Job market if we look at the Job Description, we can see the developers have been categorized as:

    • Frontend developers (Angular, React, Javascript, HTML, CSS)
    • Backend Developers (Java, Database)
    • Full Stack developers.
    • Big Data (Hadoop, spark)
    • Machine Learning
    • …

    As far as I recalled, in my early days of career back in 2005, the main categories were:

    • web designer (Dreamweavers, HTML, CSS, Photoshop)
    • PHP Developers
    • C++ Developers
    • Java, Jsp developers
    • DBAs

    These categories system has been great help to the recruiters to identify the candidates.

    But as a techincal person, I’m bit concerned about the invisible boundary which limit my mindset when I accept it unconsciously.

    For example, when I say

    I’m a Java developer

    This implies I’m saying

    I’m not a C++, C or Assembly language developer.

    So when I explore the source code in JVM, I tend to stop when native libary written in C surfaces.

    When I check the source code in Tomcat, I tend to stop when I need to check the linux implementation of connect_socket()

    There is an invisible boudary in my mind which limits my exploration and curiosity.

    And there are two striking stories I came across, which points to one wisdom in the Tech world:

    There is no boundary in technical world.

    • The creater of Spring framework, Rod Johnson is a PhD in musicology.

    • The creator of Vuejs, Evan You, majors in Art and Art History.

    I’ve been reading their stories and documentories, and these fellows have impacted the life of millions of developers, and I can’t even call them developers if I only know the education path.

    This made me think if we are letting our Education Path define who we are and limit we can or we should do?

    Probably my key thoughts are:

    1. Don’t let your education background, work experience limit who you are and what you want to explore.

    2. Don’t use these categories as the only criteria to analysis others’ potential.

    This probably perfectly callback the key essense of a scrum developer, who need to be Cross Functional.

    There, that’s my gentle reminder for myself today.
